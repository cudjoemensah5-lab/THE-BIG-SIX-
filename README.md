# THE-BIG-SIX-
# 🎓 The Big Six · UPSA Results Portal

> A private academic results portal for **The Big Six** study group at the University of Professional Studies, Accra (UPSA). Built for one purpose — making it easy for the group admin to publish exam scores and CGPA, and for each member to view their results securely.

---

## 📌 What This Is

This is a single-page web portal that lets the group admin enter, calculate, and publish examination scores for all six members of the study group. Students log in privately using their name and a unique access token to view their individual results, CGPA, subject breakdown, and leaderboard standing.

No backend server needed. All data syncs in real time through **Firebase Firestore**.

---

## ✨ Features

- **Admin Portal** — PIN-protected dashboard to enter scores, manage subjects, and publish results
- **Student Portal** — Secure login with name + token to view personal results
- **Auto CGPA Calculation** — Based on UPSA's 4.0 grading scale
- **Leaderboard** — Live group rankings with medal positions 🥇🥈🥉
- **Score Charts** — Radar chart, bar chart, and CGPA trend over semesters
- **Transcript Print** — Printable academic transcript for each student
- **WhatsApp Share** — Generate and copy result messages to send directly to members
- **Semester History** — Archive past semester results and track CGPA progress over time
- **Real-time Sync** — Results update live for students the moment admin publishes
- **Offline Fallback** — Falls back to localStorage if Firebase is unavailable

---

## 🔐 How Access Works

### Admin
- Click **Admin Portal** on the home screen
- Enter the admin PIN (default: `1234` — change this after first login)
- Enter scores, calculate results, then click **Publish**

### Students
- Click **Student Portal** on the home screen
- Enter your **full name** and your **personal access token**
- Tokens are generated by the admin and shared via WhatsApp

## ⚙️ Firebase Setup (For Admin/Dev)

1. Go to [console.firebase.google.com](https://console.firebase.google.com)
2. Open the **the-big-six** project
3. Navigate to **Firestore Database** → create the database in test mode
4. Set the following security rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /sessions/main {
      allow read: if true;
      allow write: if true;
    }
  }
}
```

5. Under **Authentication → Settings → Authorized Domains**, add your GitHub Pages URL
